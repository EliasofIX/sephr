## Repaint-loop bug (Workstream A target)

Observer profile shows renderer hot stacks reveal continuous full Blink style-recalc → layout → paint cycle while idle (even with window occluded, before the tab is scrolled or interacted with). This is the core repaint-loop bug — Sephr is triggering frame invalidation in a tight loop during normal browsing, unlike Helium or stock Chromium, causing unnecessary GPU load and thermal / battery drain.

**Expected behavior:** Idle tab (no user interaction, content not scrolling) should paint once and settle to 0.0% CPU and GPU, matching the reference 0.0% baseline. Render/GPU >0% only during active scroll, animation, or layout-affecting DOM mutation.
